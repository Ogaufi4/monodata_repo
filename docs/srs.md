# Botswana Indigenous Intelligence Data Platform

## Software Requirements Specification: Media and Translation MVP

Version 1.0

This document formalizes the media and text-translation requirements from the
original Botswana Indigenous Intelligence Data Platform product brief. The
broader implementation roadmap remains in `implementation-plan.md`.

## 1. User outcomes

A registered contributor shall be able to:

1. Upload an image from a computer or mobile device.
2. Open a mobile device camera from the image contribution form.
3. Upload an existing audio or speech file.
4. Record speech in a supported browser.
5. Stop, preview, and re-record speech before submission.
6. Submit source and target text as a translation pair.
7. Select source language, target language, category, domain, and license.
8. grant explicit consent before final submission.
9. Receive a clear error when authentication, metadata, storage, or validation
   requirements are not satisfied.

## 2. Supported media

### 2.1 Images

The MVP accepts these validated image MIME types:

- JPEG
- PNG
- WebP
- GIF
- AVIF
- BMP
- HEIC
- HEIF
- TIFF

SVG is intentionally excluded because active content in SVG files requires a
separate sanitization pipeline.

### 2.2 Audio and speech

The MVP accepts:

- WAV
- MP3
- FLAC
- AAC
- M4A/MP4 audio
- OGG
- WebM

Browser recording shall use the best supported format in this order: WebM
Opus, WebM, OGG Opus, then MP4 audio.

### 2.3 Upload limits

The default maximum object size is 500 MiB. The API shall reject unsupported
MIME types and oversized requests before issuing an upload URL.

## 3. Upload workflow

1. The contributor creates a draft contribution.
2. The web app requests a short-lived signed upload URL.
3. The browser uploads directly to Cloudflare R2 in deployed environments.
4. A development-only signed local object endpoint may be used locally.
5. The API verifies the stored object's content length and content type.
6. Only a verified object creates a `contribution_assets` record.
7. Media contributions cannot be submitted without a verified asset.
8. Upload tokens are bound to the user, contribution, storage key, MIME type,
   and expected file size.

Local object storage must never be enabled outside the development environment.
Production and preview deployments use a private Cloudflare R2 bucket.

## 4. Text translation workflow

A translation contribution shall store:

- source language
- optional source dialect
- target language
- optional target dialect
- source text
- target text
- category
- domain
- context
- notes
- synthetic-data flag
- review status
- quality score when reviewed

The target language, source text, and target text are mandatory. A translation
cannot leave draft status until its text pair and consent record exist.

## 5. Consent

Every final submission requires a versioned consent record. Contributors can
separately choose:

- AI training use
- research use
- commercial use
- open dataset release
- attribution

The interface must separately require acknowledgment of storage and review.

## 6. Security and deployment

- Public registration always creates a contributor role.
- Contributors may only modify their own draft contributions.
- Upload URLs and confirmation tokens expire after 15 minutes.
- Object keys are generated by the API and filenames are normalized.
- Stored assets remain private.
- Vercel deployments use R2 signed URLs rather than proxying large objects
  through a Vercel Function.
- Credentials and local runtime data must not be committed to Git.

## 7. Acceptance criteria

This slice is accepted when:

1. API tests cover authentication, draft validation, translation creation,
   signed upload creation, object verification, consent, and submission.
2. All Alembic migrations apply from an empty database.
3. The Next.js production build succeeds.
4. A live local image upload returns HTTP 204, confirms an image asset, and can
   be attached to a contribution.
5. A live local audio upload returns HTTP 204, confirms an audio asset, and the
   contribution reaches `submitted`.
6. A live text translation stores and returns its source and target text.
7. The audio page exposes record, stop, preview, and re-record controls in
   browsers that implement `MediaRecorder`.
